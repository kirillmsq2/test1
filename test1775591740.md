**Краткий анализ встречи**

1. Тема — внедрение QR‑платежей через сервис ERIP для клиентов банка‑бенефициара и банка‑плательщика, а также регистрация терминалов/клиентов в директории RTP.  
2. Участники — клиент (плательщик), мобильное приложение банка‑плательщика, POS‑терминал (касса), кассовое ПО (Kassa‑PO / Terminal‑PO), банковская система (банк‑бенефициар), сервис ERIP (внешний), внутренний сервис ERIP (прослойка банка), директория RTP, система анти‑фрод, система межбанковского расчёта SMP, база данных/хранилище.  
3. Описанный процесс — сканирование QR‑кода, запрос информации о платеже, проверка риска, подтверждение оплаты, перевод средств (информационный поток через ERIP, денежный — через SMP), уведомление о статусе оплаты. Также обсуждалась регистрация терминалов/клиентов в RTP через API.  

**Допущения**

| № | Что предположено | Почему |
|---|------------------|--------|
| 1 | Клиент сканирует QR‑код через мобильное приложение банка‑плательщика. | В тексте упоминается «мобильное приложение его банка». |
| 2 | Запрос на генерацию QR‑кода (динамический) отправляется от POS‑терминала в внутренний сервис ERIP, который публикует Envoy в директорию RTP. | Описана последовательность: «формируем Envoys и публикуем его в директории RTP». |
| 3 | При подтверждении оплаты мобильное приложение запрашивает статус Envoy у ERIP, который в свою очередь запрашивает у банка‑бенефициара риск‑индикатор и статус счета. | Описано, что ERIP «запрашивает у нас информацию дополнительную», включая risk‑indicator и статус счета. |
| 4 | Денежный перевод между банками проходит через SMP, если банки разные; при внутрибанковском сценарии – через BIS (внутренний процессинг). | Указано различие «межбанковская система … SMP» и «внутрибанковская – без SMP». |
| 5 | Регистрация терминала/клиента в RTP происходит через API‑метод, предоставляемый банком‑бенефициаром. | Уточнено: «регистрация … через API». |
| 6 | Все сообщения являются синхронными запрос‑ответ (→ / -->). | В тексте нет упоминаний о асинхронных колл‑бэках, поэтому использованы синхронные стрелки. |

**PlantUML‑диаграмма (основной сценарий оплаты QR‑кода)**

```plantuml
@startuml
title QR‑платёж через ERIP (основной сценарий)

' Участники
actor "Клиент (плательщик)" as Client
participant "Мобильное приложение банка‑плательщика" as MobileApp
participant "POS‑терминал (касса)" as POS
participant "Кассовое ПО (Kassa‑PO / Terminal‑PO)" as CashSoft
boundary "Внутренний сервис ERIP (прослойка)" as ERIP_Bank
boundary "Внешний сервис ERIP" as ERIP_External
database "Директория RTP" as RTP
participant "Система анти‑фрод" as AntiFraud
participant "Банк‑бенефициар" as BenefBank
participant "SMP (межбанковский расчёт)" as SMP

' == Регистрация терминала/клиента (один раз) ==
group Регистрация терминала в RTP
    BenefBank -> ERIP_Bank : RegisterTerminalAPI(терминал‑данные)
    ERIP_Bank -> RTP : POST /terminals (терминал‑данные)
    RTP --> ERIP_Bank : 201 Created
    ERIP_Bank --> BenefBank : OK
end

' == Формирование QR‑кода (динамический) ==
group Формирование QR‑кода
    POS -> CashSoft : Запрос на оплату (корзина)
    CashSoft -> ERIP_Bank : CreateEnvoy(cartInfo)
    ERIP_Bank -> ERIP_External : PublishEnvoy(envoy)
    ERIP_External -> RTP : StoreEnvoy(envoy)
    RTP --> ERIP_External : EnvoyID
    ERIP_External --> ERIP_Bank : EnvoyID
    ERIP_Bank --> CashSoft : QR‑code(URL+EnvoyID)
    CashSoft --> POS : QR‑code
    POS -> Client : Отображение QR‑code
end

' == Сканирование и подтверждение оплаты ==
group Сканирование QR‑кода клиентом
    Client -> MobileApp : Сканировать QR‑code (EnvoyID)
    MobileApp -> ERIP_External : GetEnvoyInfo(EnvoyID)
    ERIP_External -> RTP : RetrieveEnvoy(EnvoyID)
    RTP --> ERIP_External : EnvoyData
    ERIP_External --> MobileApp : EnvoyData
    note right of MobileApp : Показать сумму, получателя
    MobileApp -> BenefBank : RequestRiskIndicator(clientID)
    BenefBank -> AntiFraud : CalculateRisk(clientID)
    AntiFraud --> BenefBank : risk‑indicator
    BenefBank --> MobileApp : risk‑indicator, accountStatus
    MobileApp -> User : Подтвердить оплату?
    User --> MobileApp : Да
    MobileApp -> ERIP_External : ConfirmPayment(EnvoyID)
    ERIP_External -> ERIP_Bank : ProcessPayment(EnvoyID)
    alt Межбанковский сценарий
        ERIP_Bank -> SMP : TransferFunds(beneficiaryAccount, amount)
        SMP --> ERIP_Bank : TransferResult
    else Внутрибанковский сценарий
        ERIP_Bank -> BenefBank : InternalTransfer(account, amount)
        BenefBank --> ERIP_Bank : TransferResult
    end
    ERIP_Bank -> ERIP_External : UpdateEnvoyStatus(PAID)
    ERIP_External -> RTP : UpdateStatus(EnvoyID, PAID)
    RTP --> ERIP_External : OK
    ERIP_External --> MobileApp : PaymentStatus(PAID)
    MobileApp -> Client : Уведомление об оплате (PAID)
    MobileApp -> POS : NotifyPayment(PAID)
    POS -> CashSoft : Обновить статус чека
    CashSoft --> POS : OK
end

@enduml
```
